ApiUrlComparison Tool Guide

A comprehensive guide to comparing API responses and validating migrations.

Overview

The ApiUrlComparison tool allows you to compare the JSON or XML responses from two different API endpoints (e.g., a Legacy v1 API vs. a Modern v2 API). It highlights differences in structure, content, and data types.

Getting Started

  1. Select Comparison Type: Choose between REST (JSON) or SOAP (XML).
  2. Choose Mode:
    • Live Comparison: Compare two running API endpoints in real-time.
    • Baseline Testing: Compare a live API against a saved reference ("baseline") response.
  3. Configure Inputs: Enter the URLs for API 1 (Source) and API 2 (Target).

Using the Mock Server

Note for CloudHub Users: The embedded mock servers (ports 9091-9094) act as local simulators. When deployed to CloudHub, these ports are internal only and cannot be accessed from the public internet. They are intended for internal testing within the VPC or Worker.

You can start the Mock Server to test the tool without live APIs:

  1. Click Start Server in the top-left Mock Server panel.
  2. The status will change to RUNNING.
  3. Use the default mock URLs provided in the configuration fields (e.g., http://localhost:9091/api/v1/resource).

Advanced Configuration

Authentication

The tool supports Basic Auth and OAuth via the config.yaml file. In the Web UI, these settings are pre-loaded from the server configuration.

Test Data (Payloads)

For POST requests, the tool loads request bodies from:

  • Local Filesystem: src/test/resources/payloads
  • Packaged JAR: Internal classpath resources (fallback)
