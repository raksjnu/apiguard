API Forge Documentation

Master the art of API Comparison, Baseline Management, and Rapid Prototyping.

API Forge is a flexible, Java-based static and live analysis engine designed for Universal API Compliance. It allows you to validate migrations, compare REST/SOAP responses, and perform baseline testing with deep structural inspection.

üöÄ User Guide & Feature Overview

‚ö° Core Comparison
  • Setup: Enter Endpoint URLs (API 1 & API 2) and select the method (GET/POST).
  • Template Mode: Use {{token}} in payloads/URLs to iterate through scenarios.
  • Iteration Tokens: Define values in the tokens box (e.g., id=100;101) to run bulk tests.
  • Analysis: Click Run Comparison. Differences in JSON/XML structure are highlighted automatically.
üõ†Ô∏è Advanced Validation
  • Ignored Fields: Exclude specific JSON paths (e.g., id, meta.timestamp) in the Config tab.
  • Header Match Off: Toggle this to ignore all HTTP header differences.
  • Metadata Match Off: Ignore ephemeral differences like response time or size to focus purely on business data logic.
üìâ Baseline Engineering
  • Capture: Save any successful response as a "Golden Result" for future regression testing.
  • Replay: Re-run requests against live environments using the exact security context stored in the baseline.
  • Management: View, Compare, and Delete saved baselines directly from the Baselines tab.
üîê Security & Utilities
  • Mutual TLS: Upload PFX/JKS keystores in the Authentication tab for secure mTLS connections.
  • Messaging Bridge: Switch to JMS mode to Browse queues, Send messages, and diff asynchronous responses.
  • Reporting: Export detailed HTML comparison reports for offline analysis.

Real-time stream monitoring across multiple tabs with automatic backend session re-attachment.

üîå Unified JMS Support

API Forge now provides a first-class experience for asynchronous messaging systems. Bridge the gap between HTTP and JMS with a single tool.

Supported Providers
  • ActiveMQ: Built-in local broker support.
  • IBM MQ: Native connectivity with JAR sideloading.
  • RabbitMQ / TIBCO: Generic JNDI and Custom Factory support.
Advanced Operations
  • Bulk Testing: Iterate payloads over queues using tokens.
  • Deep Search: Filter browsed messages by payload tokens.
  • Message Comparison: Select any two messages for structural diffing.
