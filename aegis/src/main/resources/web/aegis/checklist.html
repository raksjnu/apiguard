Below is the complete catalog of validation rule types supported by Aegis. Use these types in your rules.yaml to define custom policies.

📁 File & Project Context

Rule Type Description Suggested Usage
FILE_EXISTS Verifies that specific files ensure key configurations are present. Check for security-config.xml, log4j2.xml, or README files.
PROJECT_CONTEXT Validates project structure and metadata. Enforce naming conventions on root folders or verify project types.
POM_VALIDATION_REQUIRED Ensures specific dependencies or plugins exist in pom.xml. Require mule-secure-configuration-property module.
POM_VALIDATION_FORBIDDEN Prohibits specific dependencies or plugins in pom.xml. Ban log4j 1.x or deprecated connectors.

🔍 Code & Token Analysis

Rule Type Description Suggested Usage
GENERIC_TOKEN_SEARCH Searches for presence of specific text patterns or tokens. Find "TODO" comments or specific function calls.
GENERIC_TOKEN_SEARCH_FORBIDDEN Fails if specific forbidden tokens are found. Detect hardcoded passwords (password=...) or keys.
GENERIC_TOKEN_SEARCH_REQUIRED Fails if specific required tokens are missing. Ensure copyright headers or specific annotations exist.
MANDATORY_SUBSTRING_CHECK Ensures a exact substring exists within a file. Verify specific configuration strings.
DLP_REFERENCE_CHECK Data Loss Prevention check for sensitive patterns. Scan for credit card numbers, SSNs, or API keys.

📌 XML Configuration

Rule Type Description Suggested Usage
XML_XPATH_EXISTS Verifies that an XML element exists matching an XPath. Ensure <secure-properties:config> is defined.
XML_ATTRIBUTE_EXISTS Checks if a specific attribute exists on an element. Verify api-gateway:autodiscovery has flowRef.
XML_XPATH_NOT_EXISTS Ensures an XML element matching an XPath does NOT exist. Prevent use of default HTTP ports (e.g., 8081).
XML_ATTRIBUTE_NOT_EXISTS Ensures a specific attribute is NOT present. Ensure `allowAnon` is not set to true.

⚙ Properties & Configuration

Rule Type Description Suggested Usage
GENERIC_PROPERTY_FILE Checks for patterns within `.properties` or `.yaml` config files. Validate naming or value formats in property files.
MANDATORY_PROPERTY_VALUE_CHECK Ensures a specific property key has a specific value. Enforce env=prod or specific versions.
JSON_VALIDATION_REQUIRED Ensures specific JSON paths or values exist. Validate fields in mule-artifact.json.
JSON_VALIDATION_FORBIDDEN Ensures specific JSON paths do NOT exist. Prevent forbidden fields in configuration JSONs.

🧪 Advanced Logic

Rule Type Description Suggested Usage
CONDITIONAL_CHECK Executes a check only if a "If" condition (another rule) is met. "If `pom.xml` exists, then check for `mule-maven-plugin`."
