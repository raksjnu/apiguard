üíª GUI Mode (Default)

The easiest way to use RaksAnalyzer. Launching the application defaults to opening the local web interface at http://localhost:8080.

Simply select your options in the form, choose your input source (Folder, ZIP, etc.), and click "Generate Documents".

For CI/CD pipelines or batch processing, use the CLI mode.

Basic Syntax

java -jar raksanalyzer-1.0.0.jar --cli --type [TYPE] --input [PATH] [OPTIONS]

Common Commands

# Analyze a Tibco Project from a Folder
java -jar raksanalyzer.jar --cli --type tibco5 --input /path/to/project

# Analyze a Mule Project from ZIP
java -jar raksanalyzer.jar --cli --type mule --input project.zip --input-type zip

# Using External Properties File (Mule)
java -jar raksanalyzer.jar --cli --type mule \
  --input project.jar \
  --config dev.properties

Command Options

Option Description Required
--cli Run in CLI mode (no UI) Yes (for CLI)
--type Project type: mule, tibco5, spring Yes
--input Path to project folder, ZIP, JAR, EAR, or Git URL Yes
--config External properties/config file path No
--output-type pdf, word, excel (comma-separated) No

üìÑ Configuration Files

You can supply an external configuration file using the --config option (CLI) or the file upload (GUI). This is crucial for resolving property placeholders in your code.

For Mule Projects

Upload a standard .properties file. Properties like ${http.port} will be resolved from this file, overriding project defaults.

For Tibco Projects

Upload a .substvar (XML) or .properties file. Global variables like %%SharedResources/Connections/DB/Host%% will be replaced with values from your config file.

üîß Troubleshooting

  • Port Conflict: If port 8080 is in use, start with --port 9090.
  • Missing Java: Ensure Java 17+ is installed. Check with java -version.
  • Memory Issues: For very large projects, increase heap size: java -Xmx2g -jar ...
